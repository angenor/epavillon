# Implementation Plan: Formulaire d'inscription PACO et Statistiques Admin

**Branch**: `003-paco-registration-stats` | **Date**: 2026-03-05 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/003-paco-registration-stats/spec.md`

## Summary

Enrichir le formulaire d'inscription PACO existant avec des champs démographiques (genre, profil d'âge, ville, pays, statut professionnel, organisation, consentement) stockés dans une table dédiée `paco_demographic_data`. Ajouter une rubrique admin pour visualiser les statistiques agrégées, la liste détaillée des inscrits et l'export CSV. Tout le code reste isolé dans les dossiers PACO existants.

## Technical Context

**Language/Version**: JavaScript (Vue 3.5+)
**Primary Dependencies**: Vue 3, Vue Router 4, Pinia, Supabase JS, TailwindCSS v4, Vue i18n, Font Awesome
**Storage**: PostgreSQL via Supabase (table existante `activity_registrations`, nouvelle table `paco_demographic_data`)
**Testing**: Vitest (unit), Playwright (e2e)
**Target Platform**: Web SPA (Firebase Hosting)
**Project Type**: Web application (Vue 3 SPA)
**Performance Goals**: Standard web app (formulaire < 3min, statistiques instantanées)
**Constraints**: Isolation PACO (suppression facile), i18n fr/en obligatoire, dark/light mode
**Scale/Scope**: Centaines d'inscrits attendus, 1 vue admin, 2 formulaires enrichis
